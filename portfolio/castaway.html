The Approach & Process

Castaway is an Angular JS web application that I built from concept to working prototype as my final apprenticeship project with Bloc. Although many platforms exist for streaming and subscribing to podcasts, none of them allow users to donate or comment directly through those streaming services. Castaway looks to remedy this problem. With or without an account, users can stream content, communicate with their favorite shows and network providers, and donate while they listen. Since podcasts and networks rely so heavily on donations from listeners to function, users are encouraged to give right to the source and recieve feedback in the form of better content more often.

In the process of conceiving and building this application, I learned a lot about compartmentalizing designs for reusability. Keeping the designs and the functionality simple and repeatable were the main goals of the project. By using the sheetsu API, we implemented a simple Google spreadsheets database to feed fake data into the working prototype and give a realistic experience.

User Stories

As a user I want to learn what the application can do for me.
As a user I want to search for podcasts.
As a user I want to search based on podcast title & network.
As a user I want to stream podcasts.
As a user I want to sign up using email or social media.
As a user I want to save my favorite podcasts.
As a user I want to create my own playlists of episodes.
As a user I want to subscribe to podcasts and instantly get new episodes.
As a user I want to donate to my favorite shows & networks.
As a user I want to share my favorite episodes and stations.
As a user I want to leave reviews of my favorite shows.
As a user I want to read announcements and updates from my favorite shows & networks.

Prototyping

Rather than test wireframe prototypes for this project, our small team put together a working Angular JS prototpye for this application. Using the sheetsu Google spreadsheets API, we hooked up a database and pulled in data from services attached to these spreadsheets. While sheetsu has its limitations, and can often timeout when too many calls are made, it worked perfectly for our simple prototype and getting design into a semi-functional working state.

The Logo
